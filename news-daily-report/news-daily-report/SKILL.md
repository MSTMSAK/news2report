# AI领域新闻日报生成

## 工作流程

生成日报分析报告分为5个阶段，按顺序执行：

1. **数据预处理** → 清洗、去重、补全缺失字段
2. **事件筛选** → 按影响力选取Top 3-5重要事件
3. **深度分析** → 对关键事件进行背景+影响分析
4. **趋势判断** → 提炼技术/应用/政策/资本趋势
5. **报告生成** → 按模板组装并执行质量检查

---

## 阶段1：数据预处理

### 输入数据要求

期望的结构化新闻数据格式：

```json
{
  "news_items": [
    {
      "title": "新闻标题",
      "summary": "简要摘要",
      "source": "来源媒体",
      "publish_time": "2026-07-03",
      "category": "技术/产品/政策/资本/应用",
      "tags": ["AI模型", "OpenAI"],
      "entities": ["涉及公司/人物"],
      "impact_level": "high/medium/low"
    }
  ]
}
```

### 预处理步骤

1. **去重**：相同事件仅保留最完整的条目
2. **字段补全**：缺失category的条目根据内容推断分类
3. **时序排序**：按publish_time倒序排列，优先处理最新事件
4. **影响力校准**：核实impact_level标记是否合理

---

## 阶段2：事件筛选

### 筛选标准

从预处理后的数据中选择 **3-5个最重要事件**，按以下维度打分：

| 维度 | 权重 | 说明 |
|------|------|------|
| 行业影响力 | 30% | 对整个AI行业的影响范围和深度 |
| 时效性 | 25% | 事件的新鲜度和紧急程度 |
| 代表性 | 25% | 是否代表某个重要趋势或方向 |
| 信息完整度 | 20% | 数据是否足以支撑深度分析 |

### 筛选原则

- 优先选不同category的事件，确保覆盖面
- 如果同category有多条重要新闻，合并为一个主题事件
- 必须至少包含1条技术类事件

---

## 阶段3：深度分析

### 强制覆盖要求

**所有**在"今日主要热点"中列出的事件，都必须在"重要事件深度分析"中逐一进行深度分析。不允许选择性遗漏任何热点事件。

### 分析框架（每个事件）

```
1. 事件概述（2-3句，核心事实）
2. 背景与动因（为什么现在发生？行业/公司/技术背景）
3. 直接影响（对各方的影响：公司、用户、行业、市场）
4. 深层含义（可能引发的连锁反应或范式转变）
```

### 写作要求

- **强制覆盖**：每个热点事件都必须有对应的深度分析段落，禁止遗漏
- **字数要求**：每个事件分析不少于150字
- **因果逻辑链**：每个论点必须呈现完整因果链（因为X，所以Y，可能导致Z），禁止纯描述性陈述
- **数据支撑**：使用具体数据/事实支撑论点，无数据支撑的观点需标注[待验证]
- **禁止空洞表述**：绝对禁止使用"意义重大""值得关注""影响深远"等无实质内容的表述
- **事实与推断区分**：已验证的事实与合理推断需清晰区分，不确定信息标注[待验证]
- **语言规范**：禁止使用emoji/表情符号；加粗仅用于小标题和关键术语，禁止整段加粗

---

## 阶段4：趋势判断

### 趋势分析维度

从四个方向提炼趋势洞察：

1. **技术趋势**：模型架构、算法突破、工程化进展
2. **应用趋势**：商业化落地、行业渗透、产品形态变化
3. **政策趋势**：监管动态、国际合作、合规要求
4. **资本趋势**：投融资热度、估值变化、并购整合

### 判断标准

- 趋势必须有至少2个事件/数据点支撑
- 区分"已验证趋势"和"早期信号"
- 避免对未来做过分确定性的预测，使用"可能""暗示""值得关注"等措辞

## 阶段5：报告生成

### 输出格式

加载参考模板：`references/report-template.md`

报告结构：

```markdown
# AI领域日报 —— [日期]

## 今日主要热点

### 热点1：[事件标题]
一句话概述。

### 热点2：[事件标题]
一句话概述。

### 热点3：[事件标题]
一句话概述。

（继续至热点5，如需要）

---

## 重要事件深度分析

### 1. [事件标题]

**事件概述**
...

**背景与动因**
...

**直接影响**
...

**深层含义**
...

（重复结构对每个Top事件）

---

## 趋势判断

### 技术趋势
...

### 应用趋势
...

### 政策趋势
...

### 资本趋势
...

---

## 风险与机会提示

每个风险/机会条目需包含：简述内容 + 逻辑支撑（为什么这是个风险/机会）+ 确定性评级（高/中/低）+ 预期时间窗口（短期0-6月 / 中期6-18月 / 长期18月+）

### 风险提示
- **风险1（确定性：高 | 时间窗口：短期）**：简述+逻辑支撑

### 机会提示
- **机会1（确定性：中 | 时间窗口：中期）**：简述+逻辑支撑

---

*数据来源：xxx | 报告生成时间：xxx*
```

### 质量检查

生成后对照 `references/quality-rubric.md` 执行自查，确保：

- [ ] 热点事件有3-5个
- [ ] **热点数量 = 深度分析事件数量**（零遗漏）
- [ ] 每个热点都有深度分析，且分析有逻辑链（因为X→所以Y→可能导致Z）
- [ ] 趋势判断有数据/事件支撑（每个趋势≥2个支撑点）
- [ ] 风险/机会提示有确定性评级和时间窗口
- [ ] 无空洞描述（无"意义重大""值得关注"等无内容表述）
- [ ] 无emoji/表情符号
- [ ] 无整段加粗
- [ ] 日期、数据等事实信息准确
- [ ] 不确定信息已标注[待验证]
